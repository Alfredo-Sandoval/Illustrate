# Web API

The backend is a FastAPI app in `illustrate_web/api/`.

## Start the server

```bash
pip install -e ".[web]" uvicorn
uvicorn illustrate_web.api.main:app --reload
```

Default local URL:

- `http://localhost:8000`

FastAPI docs endpoints (default):

- `http://localhost:8000/docs`
- `http://localhost:8000/redoc`

## Data flow

1. Upload or fetch a PDB to obtain a temporary `pdb_id` token.
2. Call `/api/render` with that token plus render parameters.

`pdb_id` tokens are UUID hex strings generated by the backend (`32` lowercase hex chars).

## Endpoints

### `POST /api/upload-pdb`

Upload a PDB file and get a render token.

```bash
curl -X POST http://localhost:8000/api/upload-pdb \
  -F "file=@data/2hhb.pdb"
```

Response:

```json
{ "pdb_id": "7f3e0b6f86f64c8f94fbf95ec8ce1d8a" }
```

### `POST /api/fetch-pdb`

Fetch from RCSB by 4-character ID and register it as an upload token.

```json
{ "pdb_id": "2hhb" }
```

Response:

```json
{ "pdb_id": "7f3e0b6f86f64c8f94fbf95ec8ce1d8a" }
```

Status codes:

- `200`: success
- `422`: invalid PDB ID format
- `404`: not found on RCSB
- `502`: network/upstream failure

### `POST /api/render`

Render image bytes from payload.

Required fields:

- `pdb_id`
- `rules` (can be empty)

`output_format` accepted values:

- `png`
- `image/png`
- `ppm`
- `pnm`
- `image/x-portable-pixmap`

Minimal PNG request:

```json
{
  "pdb_id": "7f3e0b6f86f64c8f94fbf95ec8ce1d8a",
  "rules": [
    {
      "record_name": "ATOM",
      "descriptor": "----------",
      "res_low": 0,
      "res_high": 9999,
      "color": [1.0, 0.7, 0.5],
      "radius": 1.5
    }
  ],
  "output_format": "png"
}
```

Response media types:

- `image/png`
- `image/x-portable-pixmap`

Status codes:

- `200`: success
- `400`: unknown `pdb_id`, unsupported format, or malformed rule payload
- `422`: invalid render parameters
- `500`: internal render failure

### `GET /api/presets`

Returns preset payloads.

Each item contains at least:

- `name`
- `pdb_path`
- `rules`
- `transform`
- `world`
- `outlines`

### `GET /api/health`

```json
{ "status": "ok" }
```

## Example: fetch then render

```bash
TOKEN=$(curl -s -X POST http://localhost:8000/api/fetch-pdb \
  -H "Content-Type: application/json" \
  -d '{"pdb_id":"2hhb"}' \
  | python3 -c 'import json,sys; print(json.load(sys.stdin)["pdb_id"])')

curl -X POST http://localhost:8000/api/render \
  -H "Content-Type: application/json" \
  -d "{\"pdb_id\":\"$TOKEN\",\"rules\":[{\"record_name\":\"ATOM\",\"descriptor\":\"----------\",\"res_low\":0,\"res_high\":9999,\"color\":[1.0,0.7,0.5],\"radius\":1.5}],\"output_format\":\"png\"}" \
  --output render.png
```

## Included web frontend

A Next.js frontend is included in `illustrate_web/frontend/`.

```bash
cd illustrate_web/frontend
npm ci
npm run dev
```

Defaults:

- frontend: `http://localhost:3000`
- API rewrite target: `http://127.0.0.1:8000` (override with `ILLUSTRATE_API_URL`)
